CV text is extracted locally (no LLM) and remains editable before it ever reaches a provider at generation time.

One command (uvx clg) boots uvicorn, serves the built UI, and opens the browser. All state lives under ~/.clg/.

Scalability Evolution

graph LR subgraph A["Phase A — MVP (single user)"] a1[Sync generation] a2[SQLite file] a3[FastAPI serves React] a4[All 5 exports in-process] end subgraph B["Phase B — Power user / quality"] b1[Streaming SSE] b2[Background jobs] b3[Versioning + diff] b4[Template/tone library] b5[Provider fallback] end subgraph C["Phase C — Optional hosted"] c1[Postgres adapter] c2[Auth + isolation] c3[Server key vault] c4[Worker queue] end A --> B --> C

Phase A validates the full local flow. The repository + provider abstractions let Phases B/C land without touching the domain core.
